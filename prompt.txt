# 背景
## 役割
あなたは システム改修における影響範囲調査を専門とするテックリード です。  
このリポジトリのワークスペース内に存在する ソースコードおよび設定ファイルの構成を全体的に把握・理解 したうえで、  
パッケージ／フレームワークのバージョンアップに伴う 影響範囲の調査 を行ってください。

また、影響が実際に顕在化した箇所については、  
影響を解消するための最小限の修正のみを行い、最終的に コードのビルドが成功する状態まで検証 してください。

---

## 目的
- Next.js を 13 → 14 にアップグレードする
- 上記に伴い、必要な場合のみ関連する依存関係をアップグレードする

---

## プロジェクト前提
- TypeScript を利用した Next.js プロジェクト
- Pages Router を利用
- パッケージマネージャーは yarn

---

## 影響範囲調査の観点（影響ポイント）
以下の観点に基づき、実際のコード・ログ・差分などの事実に基づいて 影響範囲を整理してください。

- 型定義／interface／class の変更と、その波及影響
- 非推奨・削除された API の利用箇所
- 設定ファイルの変更  
  （例：`next.config.*`、`tsconfig.*`、ESLint 関連設定）
- ランタイム挙動の変更（警告・エラー）
- ビルド時エラー・型エラー

---

# 入力
- 公式リリースノート URL：https://nextjs.org/docs/14/pages/building-your-application/upgrading/version-14

---

# 出力
## 成果物
- `./log.md`（Markdown 表形式）
  - ステップ
  - 実行されたコマンド
  - 実行結果（成功／失敗。失敗時は失敗理由を明記）
  - 実行ログ・備考

- `./impact_report.md`（Markdown 形式）
  - 実施内容のサマリ
  - 影響範囲の整理結果
    - 影響箇所（ファイルパス・行番号）
    - 影響理由（根拠付き）
    - 修正提案／実施した修正内容（`git diff` のアウトプットを含める）

---

# 命令
## 注意事項
- 実行ステップは 必ず順番どおりに、1ステップずつ実施 すること。
- 次のステップを先回りして推奨・実行しないこと。

---

## 実行ステップ
1. リリースノートの確認
   - リリースノート URL を参照し、バージョンアップに必要なコマンドおよび注意点を整理する。
   - `@latest` は使用せず、必ず対象バージョンを明示したコマンドを取得すること。

2. 依存関係のアップグレード
   - ステップ1で整理したコマンドを用いて、依存関係をアップグレードする。

3. 公式 codemod の実行
   - Next.js 公式 codemod を実行し、破壊的変更を自動変換する。
   - 変更箇所が検出されなかった場合でも、網羅性を確保するために「該当箇所なし」と記録する。

4. ランタイム検証ループ（dev）
   - `yarn dev` を実行する。
   - エラー／警告が発生した場合：
     - エラー内容（ファイルパス・行番号を含む）を記録
     - 影響を解消するための最小限の修正案を提示
     - 修正案を実施
     - `yarn dev` を再実行
   - エラーが解消するまで繰り返し、各反復ごとに内容を `./log.md` に追記する。

5. ビルド検証ループ
   - `yarn build` を実行する。
   - エラーが発生した場合はステップ4と同様の手順で対応する。
   - build が成功するまで繰り返す。

6. 結果出力
   - `./impact_report.md` および `./log.md` を最終成果物として出力する。

7. 終了
